# 三层意图识别系统技术方案 Spec

## 一、项目概述

构建一个三层渐进式的意图识别系统，用于对用户查询进行分类。三层结构按速度优先原则组织：关键词匹配最快、BERT次之、LLM兜底。

---

## 二、输入输出定义

### 输入
- `query`：用户输入的文本字符串（string）

### 输出
- `intent`：意图类别名称（string）
- `confidence`：置信度（float，0-1之间）
- `source`：识别来源（`"keyword"` | `"bert"` | `"llm"`）
- `processing_time_ms`：处理耗时（float，毫秒）

### 意图类别示例
```python
INTENT_CATEGORIES = [
    "educational",    # 教育/学习类问题
    "chitchat",       # 闲聊类
    "technical",      # 技术问题
    "command",        # 系统命令（如/help）
    "greeting",       # 问候语
    "other"           # 其他/无法识别
]
```
> 注：你可根据实际业务修改以上类别列表。

---

## 三、三层架构详细设计

### 第一层：关键词匹配

**优先级**：最高（最先执行）  
**速度目标**：< 1ms  
**触发条件**：无条件执行，匹配成功则短路返回

**数据结构**：
```python
KEYWORD_INTENT_MAP = {
    "greeting": ["你好", "您好", "hi", "hello", "嗨"],
    "command": ["/help", "/exit", "/start"],
    "educational": ["学习", "教程", "怎么", "如何", "什么是"],
    "chitchat": ["天气", "笑话", "心情"],
    "technical": ["bug", "错误", "安装", "报错"]
}
```

**匹配规则**：
- 遍历关键词列表，任一关键词在 query 中出现即视为匹配
- 优先级：越靠前的意图优先级越高（可用列表顺序控制）
- 匹配成功后返回对应的意图，confidence = 1.0

---

### 第二层：BERT 轻量模型分类

**优先级**：仅在关键词未匹配时执行  
**速度目标**：5-50ms（CPU）/ 5-20ms（GPU）  
**触发条件**：第一层未匹配

**模型配置**：
- 模型名称：`bert-base-chinese`（中文场景）/ `bert-base-uncased`（英文场景）
- 任务类型：文本分类（SequenceClassification）
- 输出类别数：根据 INTENT_CATEGORIES 长度确定
- 运行设备：优先 GPU，无 GPU 则 CPU

**输出要求**：
- 返回 top-1 意图类别
- 返回 softmax 概率作为 confidence
- confidence 阈值：需配置可调参数（建议默认 0.7）

**置信度处理逻辑**：
```
if confidence >= CONFIDENCE_THRESHOLD:
    返回 BERT 结果
else:
    降级到第三层（LLM）
```

---

### 第三层：LLM 兜底分类

**优先级**：最低，仅在 BERT 置信度不足时执行  
**速度目标**：200-800ms  
**触发条件**：BERT 输出的 confidence < 阈值

**LLM 配置**：
- 模型选择：推荐 OpenAI GPT-3.5-turbo 或本地部署的 Qwen/DeepSeek
- 调用方式：API 或本地推理
- 超时设置：3 秒
- 失败处理：超时或异常时返回 "other" 意图

**Prompt 模板**：
```text
将以下用户查询分类到指定的意图类别中。

意图类别：{INTENT_CATEGORIES 列表}

用户查询："{query}"

请只返回意图类别名称，不要有其他内容。
```

## 四、决策流程伪代码

```python
def classify(query):
    start_time = now()
    
    # 第一层：关键词匹配
    keyword_result = match_keywords(query)
    if keyword_result:
        return build_response(keyword_result, "keyword", 1.0)
    
    # 第二层：BERT
    bert_result = bert_classify(query)
    if bert_result.confidence >= CONFIDENCE_THRESHOLD:
        return build_response(bert_result.intent, "bert", bert_result.confidence)
    
    # 第三层：LLM
    llm_result = llm_classify(query)
    return build_response(llm_result.intent, "llm", llm_result.confidence)
```

---

## 五、配置参数（可调）

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `CONFIDENCE_THRESHOLD` | 0.7 | BERT 结果可接受的最低置信度 |
| `BERT_MAX_LENGTH` | 128 | BERT 输入的最大 token 长度 |
| `BERT_BATCH_SIZE` | 32 | BERT 批处理大小 |
| `LLM_TIMEOUT_SECONDS` | 3 | LLM 调用超时时间 |
| `ENABLE_LLM_FALLBACK` | True | 是否启用 LLM 兜底（可关闭以节省成本） |

## 七、额外要求（可选）

1. **日志记录**：每次分类请求记录 query、结果、来源、耗时、confidence
2. **缓存机制**：对相同 query 的 BERT/LLM 结果进行缓存（TTL 可配置）
3. **数据收集**：所有经过 LLM 处理的 case 自动保存为训练数据，用于后续微调 BERT
4. **优雅降级**：若 BERT 模型加载失败，系统应能降级到纯关键词 + LLM 模式
